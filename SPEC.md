# Daily Report Helper - 技术规格说明书

## 1. 项目概述

- **项目名称**: Daily Report Helper (日报助手)
- **项目类型**: Node.js Express 前后端分离应用
- **核心功能**: 自动从 GitLab 拉取提交记录，智能生成符合钉钉格式的日报
- **目标用户**: 需要每日提交工作日报的开发人员

## 2. 技术架构

### 后端
- **运行时**: Node.js 18+
- **框架**: Express 4.x
- **端口**: 3000 (可通过 PORT 环境变量修改)
- **存储**: JSON 文件 (config.json, history.json)

### 前端
- **技术**: 原生 HTML5 + CSS3 + Vanilla JavaScript
- **无需**: 任何前端框架或构建工具
- **响应式**: 移动端优先，适配 PC/手机

## 3. 功能模块

### 3.1 GitLab 配置管理
- 配置 GitLab 服务地址 (如 https://gitlab.com 或私有部署地址)
- 配置个人访问令牌 (Personal Access Token)
- 配置默认项目 ID
- 配置用户名/邮箱 (用于过滤本人提交)
- 配置持久化到 `config.json`

### 3.2 Git 提交拉取
- 调用 GitLab REST API v4
- 优先拉取近 24 小时内的本人提交
- 按项目、日期智能去重
- 解析提交信息，提取有效内容

### 3.3 智能降级逻辑
- 24 小时内无提交时，自动拉取 3-7 天内提交
- 合并同类工作描述
- 智能归纳总结，避免简单罗列
- 生成合理的今日工作内容

### 3.4 日报编辑
- **今日完成工作**: 自动填充，可手动编辑
- **问题与解决**: 手动输入
- **明日计划**: 手动输入
- **备注**: 手动输入
- 所有字段支持手动修改、删除

### 3.5 历史记录
- 保存历史日报到 `history.json`
- 按日期索引，可查看历史
- 支持一键清空

### 3.6 输出复制
- 生成钉钉格式日报
- 一键复制到剪贴板
- 纯文本格式，方便粘贴

## 4. API 接口

### GET /api/config
获取当前配置

### POST /api/config
保存配置
```json
{
  "gitlabUrl": "https://gitlab.com",
  "token": "glpat-xxxx",
  "projectId": "12345",
  "username": "zhangsan",
  "email": "zhangsan@example.com"
}
```

### GET /api/commits
获取 Git 提交记录
- Query: `?days=1` (默认近1天)

### GET /api/history
获取历史记录列表

### POST /api/history
保存日报到历史
```json
{
  "date": "2026-04-07",
  "content": {
    "today": "...",
    "problems": "...",
    "tomorrow": "...",
    "notes": "..."
  }
}
```

### DELETE /api/history/:date
删除指定日期的历史记录

## 8. 部署要求

### 环境变量
- `PORT`: 服务器端口 (默认 3000)
- `HOST`: 监听地址 (默认 0.0.0.0)

### 启动命令
```bash
npm install
npm start
```

### 部署平台
- Vercel: vercel.json 配置
- Render: render.yaml 配置
- 自有服务器: 直接运行

## 9. 错误处理

- GitLab 连接失败: 显示友好错误提示
- Token 无效: 提示检查配置
- 网络超时: 显示超时提示，允许重试
- 无提交记录: 自动触发降级逻辑

## 10. 安全考虑

- Token 仅存储在服务端，不暴露到前端
- 配置文件不在前端显示
- 不实现任何自动提交功能
